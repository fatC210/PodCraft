# PodCraft - 产品需求文档 (PRD) - MVP

> **版本**: MVP v0.1
> **日期**: 2026-03-23
> **产品名称**: PodCraft
> **一句话描述**: 以语音对话驱动的 AI 播客生成平台，全程语音交互，从素材采集到音频输出一站式完成。

---

## 1. 产品概述

### 1.1 产品愿景

PodCraft 是一款基于 AI 的播客生成工具，用户可通过 **全程语音对话** 与 AI 交互，完成从素材搜集、脚本编写、音色选择到最终播客音频输出的全流程。无需任何手动操作，所有步骤均通过自然语音对话完成。

### 1.2 核心技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Next.js (React) |
| 后端 | Python (FastAPI / Flask) |
| 网页抓取 | Firecrawl |
| 语音合成 (TTS) | ElevenLabs |
| 语音识别 (STT) | ElevenLabs STT |
| AI 大模型 | 用户自配（按 API 地址 + API Key 接入） |
| 部署方式 | 本地部署 |

### 1.3 目标平台

Web 应用（本地部署，浏览器访问）

## 3. 核心功能模块

### 3.1 全语音交互模式（Voice Flow）

PodCraft MVP 的核心交互方式为 **全语音对话**，用户无需任何手动操作即可完成播客创建全流程。

#### 3.1.1 语音交互流程

1. 用户语音告诉 AI 播客主题 / 方向
2. AI 根据用户描述，自动搜索相关资料（调用 Firecrawl）
3. AI 以语音播报搜索到的内容摘要
4. 用户语音决策：采纳 / 修改 / 跳过该素材 / 补充更多素材
5. 素材确认完成后，AI 语音确认生成参数（语言、角色名等）
6. AI 生成播客脚本，并语音播报脚本概要
7. 用户语音确认或要求修改脚本
8. AI 语音引导用户选择音色（试听后语音确认）
9. 生成最终播客音频

#### 3.1.2 语音交互子模式

| 模式 | 说明 |
|------|------|
| 流式实时对话 | 用户说话即时识别，AI 实时响应，体验类似语音聊天 |
| 录音后提交 | 用户录完一段话后提交，AI 处理后回复，适合嘈杂环境 |

#### 3.1.3 交互流程图

```
用户语音输入 → ElevenLabs STT 转文字 → AI 大模型处理 → 生成回复文本 → ElevenLabs TTS 语音播报
     ↑                                                                        ↓
     ←←←←←←←←←←←←←←← 用户继续语音交互 ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

#### 3.1.4 语音指令示例

| 阶段 | 用户语音示例 | AI 行为 |
|------|------------|---------|
| 主题确定 | "我想做一期关于量子计算的播客" | AI 确认主题，开始搜索相关资料 |
| 素材筛选 | "第二条不错，保留；第三条跳过" | AI 更新素材池 |
| 补充素材 | "再搜索一下量子纠缠的最新实验" | AI 追加搜索 |
| 素材通过链接获取 | "帮我抓取这个链接的内容：xxx.com/article" | AI 调用 Firecrawl 抓取链接 |
| 参数确认 | "用中文，两个角色，一个叫小明一个叫小红" | AI 设置生成参数 |
| 脚本确认 | "把开头部分改得更有趣一点" | AI 修改脚本对应部分 |
| 音色选择 | "试听第三个音色……就用这个" | AI 播放试听音色，确认后绑定角色 |
| 生成 | "好的，开始生成吧" | AI 开始合成播客音频 |

### 3.2 内容获取方式

系统支持多种内容获取方式，均通过语音指令触发：

| 方式 | 说明 | 语音触发示例 |
|------|------|------------|
| 关键词搜索 | AI 通过 Firecrawl 搜索相关内容 | "搜索量子计算最新进展" |
| 新闻热点抓取 | AI 抓取每日新闻热点 | "帮我看看今天的科技新闻" |
| 链接抓取 | 系统抓取网页内容 | "帮我抓取这个链接的内容" |
| 文本输入 | 用户口述内容作为素材 | "我直接说一段内容当素材" |

内容获取后，AI 以语音形式逐条播报。用户语音指令：
- **保留**："这条不错，保留"
- **删除**："这条跳过"
- **替换**："换一条"
- **修改**："把这条的重点改成……"

### 3.3 播客脚本生成

#### 3.3.1 生成参数（语音确认）

| 参数 | 说明 |
|------|------|
| 输出语言 | 跟随 ElevenLabs 支持的语言范围（含中文、英文及多语言） |
| 自定义角色名 | 为播客中的角色命名（如 "主持人A"、"嘉宾B"） |
| 指导提示 | 用户语音补充的风格/内容引导 |

AI 会在生成前以语音逐一确认上述参数，用户语音回复确认或修改。

#### 3.3.2 脚本编辑（语音驱动）

| 功能 | 语音交互方式 |
|------|------------|
| 整体修改 | "把整个脚本的语气改得更轻松一些" |
| 局部修改 | "把开头的介绍部分重新写一下" |
| AI 优化 | "帮我润色一下第三段" |
| 确认完成 | "脚本没问题了，继续下一步" |

### 3.4 声音配置（语音驱动）

#### 3.4.1 音色选择

- **供应商**: ElevenLabs
- **预设音色**: AI 语音播报可选音色，支持逐一试听
- **角色绑定**: 用户语音指定每个角色使用的音色

语音交互流程：
1. AI："现在为角色'主持人'选择音色，我播放几个供您选择"
2. AI 依次播放音色试听片段
3. 用户："第二个不错，就用这个"
4. AI 确认绑定，继续下一个角色

### 3.5 播客生成与输出

#### 3.5.1 生成流程

```
语音确定主题 → 语音筛选素材 → 语音确认参数 → AI生成脚本 → 语音确认/修改脚本 → 语音选择音色 → 生成播客
```

#### 3.5.2 输出格式

| 格式 | 说明 |
|------|------|
| MP3 音频 | 标准音频文件 |

#### 3.5.3 生成进度

- 实时显示生成进度条
- 支持分段预览（已生成的部分可先行试听）

### 3.6 历史记录管理

- 保存所有生成过的播客记录
- 每条记录包含：播客标题、生成时间、使用的素材、脚本内容、音频文件
- 支持删除历史记录
- 数据存储在本地（无需登录注册）

### 3.7 模型供应商配置

#### 3.7.1 配置方式

用户在设置页面添加 AI 模型供应商：

| 配置项 | 说明 |
|--------|------|
| 供应商名称 | 自定义名称（如 "OpenAI"、"Claude"、"DeepSeek"） |
| API 地址 | 模型服务的请求地址（Base URL） |
| API Key | 鉴权密钥 |

#### 3.7.2 模型发现

- 配置完成后，系统自动请求 API 获取可用模型列表
- 默认使用模型列表里的第一个模型
- 支持添加多个供应商，按需切换

#### 3.7.3 其他服务配置

| 服务 | 配置项 |
|------|--------|
| ElevenLabs | API Key（用于 TTS / STT） |
| Firecrawl | API Key（用于网页抓取与搜索） |

## 5. 系统架构

### 5.1 架构概览

```
┌──────────────────────────────────────────────────┐
│                    Frontend (Next.js)              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ 语音交互  │ │ 对话展示  │ │ 播放器   │          │
│  │   模块    │ │   模块   │ │   模块   │          │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘          │
│       └────────────┴────────────┘                 │
│                        │ WebSocket + API           │
└────────────────────────┼──────────────────────────┘
                         │
┌────────────────────────┼──────────────────────────┐
│                  Backend (Python)                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │ 语音服务  │ │ 抓取服务  │ │ 脚本引擎  │           │
│  │ElevenLabs│ │Firecrawl │ │ LLM调用  │           │
│  └──────────┘ └──────────┘ └──────────┘           │
│  ┌──────────┐                                      │
│  │ 音频引擎  │                                      │
│  │ElevenLabs│                                      │
│  └──────────┘                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │              本地存储 (SQLite / 文件系统)       │  │
│  └──────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────┘
```

### 5.2 关键技术细节

#### 语音实时对话

- 前端通过 WebSocket 连接后端
- STT：前端采集音频流 → 发送至后端 → 调用 ElevenLabs STT API
- LLM：STT 结果 → 发送至用户配置的 AI 模型 → 获取回复
- TTS：AI 回复文本 → 调用 ElevenLabs TTS API → 流式返回音频给前端播放

#### 网页抓取

- 后端调用 Firecrawl API 进行网页抓取与搜索
- 抓取结果经 LLM 提炼摘要后返回前端

#### 播客音频合成

- 后端按脚本角色分段调用 ElevenLabs TTS
- 将多段音频拼接为完整播客
- 输出 MP3 文件

### 5.3 本地存储方案

| 数据 | 存储方式 |
|------|---------|
| 用户配置（API Key、模型供应商） | SQLite / JSON 文件（加密存储敏感信息） |
| 播客历史记录 | SQLite（元数据） + 文件系统（音频文件） |
| 临时素材 | 文件系统（临时目录，可清理） |
| 脚本数据 | SQLite |

## 6. API 设计概要

### 6.1 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/voice/stream` | WebSocket | 语音实时对话（STT + LLM + TTS） |
| `/api/voice/transcribe` | POST | 录音转文字（非实时模式） |
| `/api/sources/search` | POST | 搜索关键词（语音触发） |
| `/api/sources/url` | POST | 抓取链接内容（语音触发） |
| `/api/script/generate` | POST | 生成播客脚本 |
| `/api/script/ai-edit` | POST | AI 辅助编辑脚本片段 |
| `/api/voices/list` | GET | 获取可用音色列表 |
| `/api/podcast/generate` | POST | 生成最终播客音频 |
| `/api/podcast/history` | GET | 获取历史记录 |
| `/api/podcast/{id}` | GET | 获取单条播客详情 |
| `/api/podcast/{id}` | DELETE | 删除播客记录 |
| `/api/settings/providers` | GET/POST/PUT/DELETE | 模型供应商 CRUD |
| `/api/settings/providers/{id}/models` | GET | 获取供应商可用模型 |
| `/api/settings/services` | GET/PUT | ElevenLabs / Firecrawl 配置 |

## 8. MVP 范围边界

### 包含（In Scope）

- 全语音交互创建播客（流式 + 录音两种模式）
- 内容获取：关键词搜索 / 新闻热点 / 链接抓取 / 语音口述
- 脚本生成与语音驱动编辑

- 本地历史记录
- 模型供应商自配置
- 界面国际化（中文/英文，默认中文）
- 本地部署

### 不包含（Out of Scope）

- 手动操作模式（图形界面素材管理、脚本手动编辑等）
- 播客格式选项（Deep Dive / Brief / Critique / Debate）
- 文件上传素材
- 声音克隆
- BGM 配置（用户上传 + AI 生成）
- 带封面音频输出
- 脚本版本对比
- 素材标签、分组、拖拽排序
- 用户注册 / 登录系统
- 云端同步与多设备共享
- 播客发布到第三方平台
- 多人协作编辑
- 视频播客
- 移动端适配

## 10. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| ElevenLabs API 延迟较高 | 语音对话体验卡顿 | 前端增加加载态动画；支持切换录音模式 |
| 用户配置的 LLM 模型质量参差不齐 | 脚本质量不稳定 | 提供 Prompt 模板兜底；建议用户使用主流模型 |
| Firecrawl 抓取部分网站失败 | 素材获取不完整 | 语音提示用户换个关键词或口述素材；错误提示明确 |
| 本地存储空间占用 | 长期使用后占用过多磁盘 | 提供存储清理功能；显示已用空间 |
| 音频拼接质量 | 多段 TTS 拼接不自然 | 段间插入自然停顿 |
| 纯语音交互理解偏差 | 用户意图识别不准确 | AI 主动确认关键决策；支持用户语音纠正 |
